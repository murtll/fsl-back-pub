Documentation for API

fsldev.herokuapp.com

Mappings for actions with users (starts with /user)

  • /{id} - GET - get information about user with id {id}
  • /all - GET - get information about all users
  • /add - POST - add new user
    gets JSON object like this -
    {
    "username": "[username]",
    "email": "[email]",
    "password": "[password]"
    }
    sends confirmation email to email address specified in request
    returns HTTP 200 OK if all was good or HTTP 400 BadRequest if smth went wrong
  • /confirm/{key} - GET - confirms that this key really was taken if it was and was not used yet, user with this key becomes active user
  • /me - GET - gives to client information about this user based on his token (client must be authorized)

Mappings for actions with tokens (starts with /token)

  • /get - POST - gives client an access token and a refresh token if credentials are good
    in other way returns HTTP 400 BadRequest
    gets JSON object like this -
    {
    "email": "[email]",
    "password": "[password]"
    }
  • /refresh - POST - gives client a new access token and refresh token
    if refresh token haven`t expired yet AND !!! ACCESS TOKEN HAS ALREADY EXPIRED !!!
    if access token haven`t expired yet, new tokens won`t be given

Mappings for actions with posts (starts with /post)

  • /{id} - GET - get information about post with id {id}
  • /all - GET - get information about all posts
  • /add - POST - add a new post (client must have admin role)
    gets JSON object like this -
    {
    "title": "[title]",
    "text": "[text]",
    "photoLink": "[link to photo]"
    }
    returns HTTP 200 OK if all was good or HTTP 400 BadRequest if smth went wrong

Mappings for actions with annos (starts with /anno)
